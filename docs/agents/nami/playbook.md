## 0. 현재 단계 제약 (중요)

- **DayZero B2C는 Pre-launch 단계** (MVP 개발 중, 2026-03~). 실사용 불가.
- **유일한 Activation = `dzero.run` 사전 신청** (30~50% 할인 약속).
- **가짜 후기/"써보니 좋았다" 금지**. 크리티컬 원칙.
- 콘텐츠 톤은 **"곧 나올 솔루션 — 지금 미리 등록하면 혜택"**이다.

## 1. 미션

나미 팀은 **무풍지대를 탈출할 바람**을 만든다.
2026년 6월 전까지 DayZero B2C의 **사전 신청 총량**을 최대한 끌어올리는 것이 유일한 성공 기준이다.

> ✅ 좋은 콘텐츠 = 이번 주 안에 `dzero.run` 사전 신청을 만든다 (또는 브랜드 인지 → 다음 노출 시 전환 확률을 높인다)
> ❌ 나쁜 콘텐츠 = 좋아 보이지만 사전 신청·브랜드 인지·둘 다 못 만드는 콘텐츠

## 2. 타겟 독자

### 🎯 Primary — "지쳐 있는 역직구 셀러"

Qoo10 JP 이미 운영 중, 수동 관리로 하루 3~6시간 소진, 품절/역마진 경험 있음. 위탁·구매대행 1년+, 연매출 5천만~5억, 1~3인 팀.

**Pain Point (콘텐츠에 반드시 녹여야 함)**:
1. 품절 뒤늦게 인지 → Qoo10 JP 계정 경고
2. 원가 올랐는데 모르고 팔아 역마진
3. 번역 외주 월 30만원+
4. 상품 등록 하루 10개가 한계

### 🎯 Secondary — "역직구 진입 고민 중인 국내 셀러"

쿠팡·스마트스토어에서 성장 한계 느끼는 셀러.

### 🎯 Tertiary — "온꿈사 생태계 학습자"

온꿈사(투플렉스) 유튜브·교육 시청자 중 아직 시작 못 한 예비 셀러.

### 비-타겟 (Out of scope)

대기업 수출 담당, 일본 로컬 판매자, 쇼피·이베이 단독 셀러, "AI 구경꾼" 일반 독자.

### 4a. Threads 세부 전략

**목표**: **브랜드 인지도 + 팔로워 축적 + 사전 신청 간접 유도**. 광고로 인식되면 알고리즘 페널티 → 도달 급감. 이 채널은 **인사이트 주는 계정**으로 포지셔닝.

#### ① 레퍼런스 수집 소스 = 업종 무관 ✅

- **마케팅·생산성 SaaS·창작자·스타트업 창업자** 계정 등 **다양한 업종**의 잘된 포스트.
- 업종이 아니라 **후킹·포맷·사람들이 멈추는 구조**를 배운다.
- 이 수집·정제는 상디(시장 정보)에게 의뢰 가능한 크로스팀 작업.

#### ② 레퍼런스 활용 범위 = 레퍼런스별 전략

| 레퍼런스 타입 | 따오는 범위 |
|---|---|
| 후킹·서두 구조 (예: "3년 장사하며 알게 된 것") | **포맷만** 참고 → 우리 역직구 앵글로 채움 |
| 주제 앵글 (예: "대부분의 셀러가 놓치는 것") | **앵글까지** 참고 → 같은 각도를 역직구에 적용 |
| 구체 문장·표현 | **그대로 쓰지 않음**. 각색하되 출처 인용이 필요하면 명시 |

#### ③ 상품 노출 강도 = 10:1 (소프트셀)

- **10개 중 9개**: 순수한 셀러 인사이트·역직구 업계 팁·시장 관찰 (DayZero 직접 언급 없음, 단 "INVAIZ / 나미" 브랜드 시그널은 은은히)
- **10개 중 1개**: DayZero 직접 언급 (기능 소개 or 사전 신청 유도)
- 이 비율은 **사람들의 반응을 보며 조정**한다 (v0.2에서 재검토).

#### ④ 계정 운영 주체 = DayZero 공식 톤

- 공식 계정이지만 **딱딱한 기업 톤 X**. 스레드에 잘 먹히는 **캐주얼한 관찰자·조언자 톤**.
- **각 업종 잘나가는 계정들의 말투·문장 구조를 수집**해 별도로 분석 (이것도 레퍼런스 수집의 일부).
- 자사 "나미" 캐릭터 톤은 **내부 노션**과 디스코드에서만 사용. 외부 스레드는 DayZero 공식 톤.

#### ⑤ 피할 주제 (논란 회피)

- **절대 금지**: 정치, 젠더, 종교, 특정 집단 비하, 민감한 사회 이슈 편승
- 그 외는 **검수자(담당자)가 발행 전 최종 체크**. 애매하면 스킵.

#### ⑥ 발행 운영

| 항목 | 기준 |
|---|---|
| 발행량 | 주 5~10건 (일 1건 전후) |
| 길이 | 스레드 1~5개 포스트 체인 (첫 포스트로 후킹) |
| CTA | 10개 중 1~2개에만. 나머지는 프로필 링크(`dzero.run`)에 맡김 |
| 해시태그 | v0.2에서 확정. 현재는 최소 사용 |

#### ⑦ 자동 레퍼런스 수집 파이프라인 (착수 확정)

**시간표 (맥미니 cron)**
- 03:00 seed 계정 공개 포스트 크롤 (Playwright)
- **06:00 스코어링 + 큐레이션 (TOP 10 선정)**
- **07:00 노션 페이지 자동 생성 + 디스코드 `#콘텐츠팀-나미` 배달**

**Engagement score**: `likes + reposts×3 + replies×2`
**분포 제약**: 업종 쏠림 방지(한 업종 최대 3건) / 전일 중복 작성자 최대 1건
**언어 기본값**: 한국어 70% + 영어 30%
**업종 분포 기본값**: 마케팅·브랜딩 30% / 생산성·SaaS 20% / 창업·1인기업 20% / 창작자 15% / 커머스 10% / 기타 5%

**학습 루프**: 나미가 활용한 레퍼런스 → `활용됨` 마킹 → 해당 작성자 가중치↑. 2주간 활용 0건 seed는 후순위.

**착수 순서**
1. [ ] seed 리스트 v0 구축 (50~100개, 나미가 외부 리서치)
2. [ ] threads.net 공개 페이지 스크래핑 실측 (수치 노출·robots.txt·로그인 요구 여부)
3. [ ] 스크래퍼·태거·스코어러 구현 (`src/agents/nami/teams/research/collectReferences.ts`)
4. [ ] 지식 베이스 DB 태그 필드 확인·보완
5. [ ] cron 등록 (03:00 수집 / 06:00 큐레이션 / 07:00 배달)
6. [ ] 3일 시범 운영 후 임계값·분량 튜닝

### 4b. Blog (SEO) 세부 전략 — 스켈레톤

**목표**: 타겟이 **실제로 궁금해하는 질문**에 친절히 답해서 **검색 유입**을 장기적으로 쌓고, 말미에 사전 신청 CTA로 전환.

#### ① 질문·관심사 소스 = 복합 (상디가 선 수집)

- 네이버·구글 검색어 (실제 검색량)
- 네이버 카페·디시·오픈채팅방 Q&A (역직구 관련)
- 온꿈사 유튜브 댓글·시청자 질문
- 상디 일일 브리핑에서 감지한 이슈
- → 상디가 이들을 주기적으로 수집·정제해 나미에게 전달

#### ② 포지셔닝 믹스 = 입문자 가이드 + 사례·경험 공유

| 포맷 | 비율 (초기) | 예시 |
|---|---|---|
| 입문자 가이드 (A-to-Z) | ~50% | "Qoo10 JP 시작하기 2026 완전판" |
| 사례·경험 공유 | ~50% | "월 5백만원 구매대행 셀러가 자동화로 바꾼 것" |

⚠️ Pre-launch 제약상 "DayZero를 써서 해결했다" 류의 **가짜 사례 금지**. 대신 "이런 페인은 이렇게 해결할 수 있는데, 마침 곧 DayZero가 이걸 자동으로 해줄 예정" 식의 정직한 구조.

#### ③ CTA 강도 = 글 끝 1회 (A)

- 본문 중간은 순수 가이드. 압박 없음.
- 글 말미에 자연스럽게 **"이 과정을 자동화해주는 DayZero가 곧 나옵니다 → 사전 신청 30~50% 할인"** 1회.

#### ④ 키워드 전략 = **사전 조사 필요** (Phase 2)

- 현시점 미확정. 스레드 우선 운영하면서 **상디가 역직구 관련 검색어·검색량을 수집**하는 별도 태스크로 먼저 돌린다.
- 수집 완료 후 v0.2 에서 1~3순위 키워드를 확정.

#### ⑤ 분량·형식 — 미확정

글자수, 이미지 수, 내부 링크 규칙은 키워드 세트가 잡힌 뒤 결정.

## 7. 운영 규칙 & 금기

**생성 플로우** (스레드 기준)
```
상디 레퍼런스 풀 업데이트
  → 나미: 후킹 템플릿 선택 + 역직구 앵글 매핑
  → runClaude()로 초안 생성 (DayZero 공식 톤)
  → 노션 콘텐츠 DB 저장 (상태: 초안)
  → 담당자 1인 검수 (논란·Pre-launch 위반 체크)
  → 스레드 발행
  → 반응 데이터 수집 → 다음 주 반영
```

**금기**
- ❌ 가짜 후기·"써보니 좋았다"
- ❌ 경쟁사 실명 비방
- ❌ 정치·젠더·종교·사회 이슈 편승
- ❌ 통계·숫자 미검증 인용
- ❌ 여러 채널에 같은 원고 그대로 복붙
- ❌ robots.txt 미확인 크롤링
- ❌ Claude Code CLI 외 SDK 호출 / ANTHROPIC_API_KEY 사용

## 9. 다른 에이전트와의 연결

- **상디 → 나미**:
  - 업종 무관 스레드 레퍼런스 수집·정제 (핵심 의존성)
  - 역직구 검색어·카페·오픈채팅 질문 수집 (블로그 준비)
  - 경쟁사·시장 동향 일일 브리핑 → 콘텐츠 소재
- **우솝 → 나미**: 새로 지원된 쇼핑몰 정보 → "지원 쇼핑몰 확장" 포스트 소재
- **조로 ← 나미**: 발행된 스레드·블로그를 콜드메일 레퍼런스로 활용
- **초퍼 → 나미**: (런칭 후) 실사용 UT 인사이트 → 후킹 카피
- **나미 → 루피**: 주간 콘텐츠 성과(유입·CTR·사전 신청 기여) 요약을 월요일 브리핑 전에 전달
- **루피 → 전체**: 매주 월 09:00 `#대장-루피` 채널에 사전 신청자 DB 기반 **무풍지대 탈출 주간 브리핑** 발송 — 나미 콘텐츠가 바람을 만들었는지 여기서 판정됨
