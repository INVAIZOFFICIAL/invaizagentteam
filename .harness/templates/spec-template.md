# 제품 스펙 (Product Spec)

> Planner가 작성하는 문서. 사용자의 요청을 완전한 제품 설계로 확장한다.
> 이 문서가 승인되면 Generator의 작업 범위를 정의하는 **최상위 계약**이 된다.

## 1. 프로젝트 개요

### 한 줄 요약
원피스 캐릭터 인격을 가진 6인 AI 에이전트(루피·나미·조로·우솝·상디·초퍼)가 디스코드를 인터페이스로, 노션을 결과 저장소로 삼아 INVAIZ DayZero B2C 확대를 위한 마케팅·리드수집·DOM분석·시장정보·사용자리서치 5개 업무 도메인을 Claude Code + cron으로 24/7 자동화하는 멀티 에이전트 시스템.

### 배경 & 동기

**문제점**:
- 수동 리드 수집에 회당 3~5시간 소요 — 영업팀 리소스의 상당 부분이 단순 검색·정리에 소진됨
- UT 인사이트 정리 및 수정안 도출에 건당 8시간 이상 소요 — 주 1회 이상 UT 사이클을 돌리는 것이 구조적으로 불가능
- 새 소싱처 DOM 분석 시마다 개발자가 직접 수작업 스펙 파악 → 개발 착수 지연
- 체계적인 콘텐츠 생산 파이프라인 부재 → Qoo10 JP 유기적 유입 확대에 명확한 한계
- 시장 정보·경쟁사 동향 탐색이 반복적 수동 작업 — 정보 대응 속도 저하

**해결 방식**:
- 캐릭터 인격 기반 6인 AI 에이전트 팀이 각자의 도메인을 전담 자동화
- 디스코드 채널/DM으로 업무 지시 → 에이전트가 Claude Code 실행 → 노션에 결과 적재하는 통일된 워크플로우
- cron 스케줄러로 반복 작업 무인 운영, 이상 감지 시 루피(대장)가 중재

### 대상 사용자

| 페르소나 | 설명 | 핵심 니즈 | 기술 수준 |
|---------|------|---------|---------|
| **마케팅 담당자** (주) | Qoo10 JP 콘텐츠 기획·업로드 담당 | 코딩 없이 SEO 콘텐츠 대량 생성, 경쟁사 벤치마킹 자동화 | 비개발자, 디스코드 친숙 |
| **영업·BD 담당자** (주) | DayZero 신규 고객사 발굴 담당 | 타깃 리드 자동 수집, 개인화 콜드메일 발송 | 비개발자, 기본 설정 가능 |
| **개발자** (주) | RPA·크롤러 개발 담당 | 새 소싱처 DOM 구조를 스펙 문서로 즉시 전달받음 | 개발자, CLI 사용 가능 |
| **리서치·PM** (보) | UT 설계·분석·수정안 도출 담당 | UT 사이클 주 1회 이상으로 가속, Atomic Nugget 클러스터링 자동화 | 비개발자, 분석 툴 친숙 |
| **팀장·경영진** (보) | DayZero B2C 전략 의사결정자 | 시장·경쟁사 일일 브리핑 자동 수신 | 비개발자, 노션 친숙 |

### 성공 기준

- [ ] 6인 에이전트 각각이 디스코드 DM 및 전용 채널에서 정상 응답 (Sprint 1 완료 기준)
- [ ] 나미가 Qoo10 JP 경쟁사 상품 데이터를 자동 크롤링 후 노션에 적재 (Sprint 2 완료 기준)
- [ ] 조로가 멀티 소스에서 리드를 수집·스코어링하고 개인화 콜드메일을 자동 발송 (Sprint 2 완료 기준)
- [ ] 우솝이 지정 URL의 DOM 트리를 분석해 RPA 스펙 문서를 30분 이내 자동 생성 (Sprint 3 완료 기준)
- [ ] 상디가 관세청·경쟁사·오픈채팅방 데이터 기반 일일 브리핑을 매일 오전 9시 자동 발송 (Sprint 3 완료 기준)
- [ ] 초퍼가 UT 시나리오 설계→Atomic Nugget 클러스터링→수정안 생성 사이클을 주 1회 이상 완주 (Sprint 4 완료 기준)
- [ ] 루피가 에이전트 간 방향 충돌을 감지하고 자동 중재 메시지를 디스코드에 발송 (Sprint 4 완료 기준)
- [ ] 전 에이전트 노션 연동 완료 및 결과 페이지 자동 생성 (Sprint 5 완료 기준)

## 3. 기술 아키텍처 (고수준)

### 기술 스택

| 영역 | 선택 (제안) | 이유 |
|------|------------|------|
| AI 엔진 | Claude Code (Anthropic API, claude-sonnet-4-6) | 에이전트 핵심 추론·생성 엔진, 대화형 태스크 실행 |
| 스케줄러 | cron (node-cron 또는 시스템 crontab) | 반복 업무 무인 실행, 시간 기반 트리거 |
| 디스코드 연동 | Discord.js v14 | 채널 라우팅·DM·슬래시 커맨드 처리 |
| 노션 연동 | Notion API (공식 SDK) | 결과 페이지 자동 생성, 데이터베이스 적재 |
| 헤드리스 브라우저 | Puppeteer | DOM 트리 분석, 동적 페이지 크롤링 |
| 런타임 | Node.js 22 + TypeScript | 타입 안전성, 비동기 I/O 성능 |
| 패키지 관리 | pnpm | 빠른 설치, 모노레포 워크스페이스 지원 |
| 테스트 | Vitest (단위) + Playwright (E2E) | 빠른 피드백, 실제 DOM·API 시뮬레이션 |
| CI/CD | GitHub Actions | 자동 테스트, 자동 배포 |
| 로�� | winston + Notion 로그 DB | 에이전트 실행 이력 중앙 집계 |

## 6. 범위 밖 (명시적으로 하지 않을 것)

- **외부 사용자 접근** — 이유: 이 시스템은 INVAIZ 내부 직원 전용이며, 외부 고객 대상 배포는 고려하지 않음
- **실시간 음성·화상 처리** — 이유: 디스코드 음성 채널 통합은 현재 업무 자동화 범위 외
- **자체 웹 대시보드** — 이유: 노션이 결과 저장소이며, 별도 웹 UI 개발은 Nice-to-have 이후 단계
- **에이전트 자율 학습·파인튜닝** — 이유: Claude API 기반으로 충분하며, 모델 커스텀은 초기 버전 범위 외
- **Qoo10 JP 외 해외 플랫폼 콘텐츠 자동 업로드** — 이유: 콘텐츠 생성만 담당하며, 실제 플랫폼 업로드는 담당자가 수동 진행
- **인사/회계/법무 업무 자동화** — 이유: 현재 스코프는 마케팅·리드·DOM·시장정보·리서치 5개 도메인만
